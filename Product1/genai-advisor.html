GenAI App Rationalization Advisor – Prototype

AI decision-support for app portfolio review

Executive Summary

The GenAI Application Rationalization Advisor is an early-stage prototype of an AI-enabled decision-support product designed to improve how enterprises evaluate and optimize their application portfolios.

It applies structured portfolio data and a defined decision workflow to support retain, retire, modernize, and consolidate decisions. The prototype generates explainable recommendations using structured inputs and LLM-based analysis, while maintaining human accountability through validation and review.

This implementation represents an early-stage product designed to validate architecture, workflow design, and business value prior to enterprise deployment.

Overview

Based on my experience as an Application Portfolio Manager at GE Vernova, this product addresses the complexity of managing large enterprise portfolios, including redundant systems, technical debt, and risk exposure.

It reduces manual analysis by structuring decision inputs and applying AI to generate consistent, explainable recommendations, while ensuring that final decisions remain with the business.

Key features

  • AI-Assisted Decision Support: Generates structured recommendations grounded in application data and business context.
  • Explainable Outputs: Provides rationale behind each recommendation to support validation and stakeholder trust.
  • Risk & Redundancy Detection: Identifies legacy systems, overlapping capabilities, and potential compliance concerns.
  • Portfolio-Level Reporting: Produces structured outputs for leadership review and planning.
  • Scalable Design: Designed to support analysis across large enterprise portfolios and evolving data inputs.

Target users

  • CIOs, Application Portfolio Managers, and Enterprise Architects (for evaluation/demo purposes)
  • Application Owners
  • Finance and Procurement Teams
  • Security and Compliance Teams

Business value

  • Improves Decision Consistency: Standardizes how applications are evaluated across the portfolio.
  • Reduces Manual Effort: Accelerates analysis that is typically fragmented and spreadsheet-driven.
  • Supports Transformation: Enables better planning for cloud migration, modernization, and consolidation initiatives.
  • Increases Transparency: Provides clear rationale behind recommendations for stakeholder alignment.

Human + AI Decision Model

  • AI generates recommendations and supporting rationale based on structured data
  • Users review, validate, and approve decisions before action is taken
  • Ownership remains with the application portfolio manager or business stakeholder
  • Designed to prevent over-reliance on AI and ensure accountable decision-making

Technology overview

The system integrates structured portfolio data with generative AI using a retrieval-based approach to produce grounded, explainable recommendations.

  • Architecture: Data ingestion, retrieval, LLM-based analysis, and structured output generation
  • Inputs: Application inventory, cost, usage, risk, and ownership data
  • Outputs: Disposition recommendations with rationale and portfolio-level reporting

MVP scope

  • Upload CSV → generate recommendations → export report
  • Generate AI-assisted recommendations for retaining, retiring, modernizing, or consolidating applications
  • Export a portfolio-level report demonstrating recommendations and rationale

Roadmap (Planned)

  • Feedback Loops: Capture user validation on AI-assisted recommendations to improve trust and learning
  • Governance Guardrails: Protect critical applications, require human approval, and show confidence scores to ensure compliance
  • CMDB Integration: Connect with enterprise CMDBs to enable richer, automated portfolio analysis
  • Persona-Specific Prompts: Provide tailored AI guidance for CIOs, Application Owners, Finance, and Security teams
  • Conversational AI Interface: Enable interactive, question-driven exploration of portfolio insights
  • Embedding Similarity & Semantic Search: Detect functionally similar applications even if names differ
  • Build trust, ensure compliance, and unlock deeper insights for smarter rationalization

How the Product Works

A structured enterprise decision-support system combining AI-generated recommendations, human validation, and measurable business outcomes.

Decision Workflow

Portfolio Data
→
AI Analysis
→
Human Review
→
Decision
  • Decision: retain, retire, modernize, or consolidate
  • AI role: generate recommendation and rationale
  • Human role: validate context, review risk, approve final decision
  • Outcome: faster and more consistent portfolio evaluation

Architecture Overview

Input Layer: application inventory, cost, usage, lifecycle, risk, ownership
Retrieval Layer: portfolio records and contextual data prepared for analysis
AI Reasoning Layer: generates disposition recommendation with supporting rationale using retrieved portfolio context
Review Layer: human validation and approval before action
Output Layer: structured recommendation, rationale, and portfolio-level report

The product is designed as a retrieval-based decision-support system where recommendations are grounded in structured portfolio data and validated through human review before action.

Evaluation Model

  • False Positives: AI recommends action where the application should be retained
  • False Negatives: AI fails to flag an application that should be retired, modernized, or consolidated
  • Consistency: similar applications receive similar recommendations
  • Rationale Quality: recommendations are understandable and reviewable

Business Impact (SQDC)

  • Safety: reduce risk from unsupported or high-exposure applications
  • Quality: improve consistency and transparency of portfolio decisions
  • Delivery: reduce manual analysis time and accelerate review cycles
  • Cost: improve investment allocation and reduce legacy spend
